Per-check 0% commission — UI options

Same data either way (buyer_checks.commission_rate ∈ {0, 5, 10}). Just two ways to expose the third option in the existing per-row check editor.

Option A · 3-state segmented pill

$2,450 @ 10% · $780 @ 5% · $320 @ 0% (store purchase)
Pros: Mutually exclusive by construction — can't accidentally tag a check as both 5% and 0%. One visual element. Mobile-friendly.
Cons: A row is taller (pill is wider than a single checkbox). Harder to skim a long list — every row shows three pills.

Option B · Keep the 5% checkbox, add a 0% checkbox next to it

Pros: Smallest visual change from today (5% checkbox is already in production). Compact rows. Familiar to existing users.
Cons: Two checkboxes that should be mutually exclusive — need to handle the case where both get ticked. Either auto-uncheck the other or grey it out. More logic + a small UX gotcha. Default state of "both unchecked" still means 10% (implicit) — slightly less self-explanatory than Option A's segmented pill where the active state is always visible.

My recommendation

Option A. The mutual-exclusion is enforced by the control itself — no logic to write, no edge case where a row ends up flagged 5% AND 0%. The visual cost (slightly wider) is worth the correctness guarantee, especially with money on the line. Mobile cell where space is tight is the only place I'd reconsider — the pill could be stacked vertically there.
