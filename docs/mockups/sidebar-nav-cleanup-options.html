Pros: reuses existing infra. Visible row count drops by ~half. Cons: requires one extra click to reach a sibling page.

Like Slack / VS Code / Linear (collapsed mode)
64px-wide rail, icons only. Labels appear on hover via tooltip, OR the user clicks a "pin open" button to expand to the full 220px sidebar.
Massive content-area reclaim — the main view gets 156px wider.
Watch out
Discoverability dips for new users. Mitigate with a first-run tooltip walkthrough OR keep labels visible on the sections you most want signposted.
Pros: dramatic visual breathing room. Optional wide/narrow toggle preserves access. Cons: requires picking great icons for all 16 items + tooltip system.

Default-collapse what the role rarely uses
For buyers: "Admin" section starts collapsed (they almost never click into it). Anyone can still expand.
For partners: Financials defaults open. For accounting: only their 4 items visible.
Tiny diff
~15 lines in Sidebar.tsx: when first computing collapsed-set state, seed it based on role.
Pros: nearly-free implementation. Different roles see a sidebar tuned to them. Cons: doesn't help users who DO use admin items often.
My pick if you want a sequence:
  • Now: ① (rename/dedupe) + ⑥ (role-aware collapse) — both pure-UX, ship in one PR
  • Next: ⑤ (smart pinned/favorites) — gives the sidebar personality
  • Big swing: ② (Cmd+K palette) — pays off forever once it exists; opens up much more aggressive sidebar trimming later
